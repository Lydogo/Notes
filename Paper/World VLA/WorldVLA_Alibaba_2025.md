# WorldVLA：迈向自回归动作世界模型

> 原标题：WorldVLA: Towards Autoregressive Action World Model
> 作者：Jun Cen, Chaohui Yu, Hangjie Yuan, Yuming Jiang, Siteng Huang, Jiayan Guo, Xin Li, Yibing Song, Hao Luo, Fan Wang, Deli Zhao, Hao Chen
> 发表：arXiv, 2025
> 链接：https://arxiv.org/abs/2506.21539
> 代码：https://github.com/alibaba-damo-academy/WorldVLA

---

## 一句话说清楚

让机器人既能"做动作"又能"想象未来画面"，把动作模型和世界模型统一到一个自回归架构中，两者互相增强——理解环境物理规律帮助做出更好的动作，而做动作的经验又反过来帮助更准确地预测未来画面。

---

## 先搞懂背景

**机器人操控面临什么问题？** 我们希望机器人看到桌上的物体、听到语言指令（比如"把碗放到炉子上"），就能自主完成操作。核心挑战是：机器人需要同时"看懂"场景和"做对"动作。

**VLA 模型是什么？** Vision-Language-Action（视觉-语言-动作）模型是近年来机器人领域的主流方案。它的思路是：拿一个已经在海量图文数据上训练好的多模态大语言模型（MLLM），给它加上一个"动作输出头"，让它根据图像和语言指令生成机器人动作。代表工作有 OpenVLA、π0 等。VLA 的优势在于继承了大模型的视觉理解和推理能力，泛化能力强。但它有一个关键短板：**动作只作为输出，从不作为输入被理解**。也就是说，模型只会"做"动作，但不真正"理解"动作会带来什么后果。

**世界模型（World Model）是什么？** 世界模型走的是另一条路：给定当前画面和要执行的动作，预测"执行这个动作后环境会变成什么样"。它本质上是在学环境的物理规律——推了碗，碗会往前滑；抓起瓶子，瓶子会离开桌面。代表工作有 iVideoGPT 等。但世界模型有自己的软肋：**它只能预测画面，不能直接输出动作**，所以不能独立控制机器人。

**痛点在哪？** 动作模型不理解物理世界，世界模型不生成动作——两者各有一半能力。能不能把它们合二为一，让一个模型同时具备"做动作"和"想象未来"的能力？这就是本文要解决的问题。

## 二、核心贡献

1. **提出 WorldVLA**：第一个基于离散自回归架构的统一动作-世界模型。在一个 LLM 框架内同时实现动作生成和未来画面预测，两者互相增强。
2. **发现并解决动作块生成的误差累积问题**：揭示了在自回归模型中连续生成多个动作时，性能会严重下降（成功率下降 10%~50%）。原因是大模型在动作领域的泛化能力有限，前序动作的错误会传导到后续动作。
3. **提出动作注意力掩码策略（Action Attention Mask）**：通过屏蔽前序动作对当前动作的影响，让每个动作只依赖视觉和文本输入，有效打断误差传播链，显著提升动作块生成性能（提升 4%~23% 成功率）。

## 三、方法原理

### 3.1 整体框架

**一句话概括数据流：** 图像 + 文本指令 + 动作 → 统一的离散 token 序列 → 同一个 LLM 自回归预测 → 输出动作 token（动作模型模式）或输出图像 token（世界模型模式）。

整体架构如下：

```
┌─────────────────────── WorldVLA ───────────────────────┐
│                                                         │
│  [动作模型模式]                                          │
│  输入: 文本指令 token + 历史图像 token                    │
│  输出: 动作 token（7 维 × K 步）                         │
│                                                         │
│  [世界模型模式]                                          │
│  输入: 文本提示 token + 当前图像 token + 当前动作 token    │
│  输出: 下一帧图像 token                                  │
│                                                         │
│  共享: 同一个 LLM backbone (Chameleon)                   │
│  共享: 统一的 token 词表 (图像 8192 + 动作 256 + 文本)    │
└─────────────────────────────────────────────────────────┘
```

关键设计思想：所有模态（图像、文本、动作）都被离散化为 token，共享同一个词表，在同一个自回归 LLM 内训练。这让动作模型和世界模型不需要两个独立网络，而是同一个模型的两种"用法"。

### 3.2 关键技术细节

#### 三种 Tokenizer：把一切变成 token

**直觉：** 要让一个 LLM 同时处理图像、文本和动作，首先得把它们都变成 LLM 能理解的离散 token。

- **图像 Tokenizer**：使用 VQ-GAN，将 256×256 的图像压缩为 256 个离散 token，512×512 的图像压缩为 1024 个 token。码本大小 8192，压缩率 16 倍。
- **动作 Tokenizer**：将连续的 7 维机器人动作（3 个位移 + 3 个角度 + 1 个夹爪状态）各自离散化为 256 个 bin 中的一个。每步动作产生 7 个 token。
- **文本 Tokenizer**：BPE 分词器，词表大小 65,536（包含了 8192 个图像 token 和 256 个动作 token）。

**为什么这么做？** 统一词表是关键——它让不同模态的 token 可以在同一个 LLM 中无缝交互，无需额外的跨模态对齐模块。

#### 动作模型的输入输出格式

动作模型接收文本指令和历史图像，输出动作序列：

$$\text{[BOS]}\{text\}\text{[BOI]}\{image\}^M\text{[EOI][EOS][BOA]}\{action\}^K\text{[EOA][EOS]}$$

其中 $M$ 是输入图像数（默认 2 帧），$K$ 是输出动作步数（chunk size，默认 5 或 10）。只对动作 token 计算损失 $L_{\text{action}}$。

#### 世界模型的输入输出格式

世界模型接收当前图像和动作，预测下一帧图像：

$$\text{[BOS]}\{text\}\text{[BOI]}\{image\}\text{[EOI][BOA]}\{action\}\text{[EOA][EOS][BOI]}\{image\}\text{[EOI][EOS]}$$

预测过程重复 $N$ 次（默认 $N=1$），只对生成的图像 token 计算损失 $L_{\text{world}}$。

注意一个设计选择：世界模型**不需要任务指令**——因为给定了具体动作，下一帧就被唯一确定了，不需要知道"为什么做这个动作"。

#### 动作注意力掩码（Action Attention Mask）——最核心的创新

**直觉：** 自回归生成多个动作时，后面的动作会"参考"前面的动作。但大模型在动作领域的泛化能力远不如在文本和图像领域（因为预训练时几乎没见过动作数据），所以前面动作的错误会像多米诺骨牌一样传导下去，越到后面错得越厉害。

**解决方案：** 修改注意力掩码，让每个动作 token **只能看到文本和图像 token，看不到前面的动作 token**。这样每个动作都是独立地基于视觉信息生成的，切断了误差传播链。

对比三种掩码：
- **(a) 默认因果掩码**：每个 token 能看到它之前的所有 token（包括之前的动作）→ 误差累积严重
- **(b) 本文提出的动作掩码**：动作 token 之间互相不可见，每个动作只依赖图像和文本 → 阻断误差传播
- **(c) 世界模型掩码**：沿用标准因果掩码，因为世界模型的预测确实需要依赖前序动作

**为什么世界模型不需要这个掩码？** 因为世界模型的输入包含真实动作（ground truth action），不存在前序预测错误传播的问题。只有动作模型在推理时会自己生成动作，才面临误差累积。

### 3.3 训练与优化

**混合训练：** 动作模型数据和世界模型数据混合训练。总损失函数为：

$$L = L_{\text{action}} + \alpha L_{\text{world}}$$

其中 $L_{\text{action}}$ 和 $L_{\text{world}}$ 都是交叉熵损失，$\alpha = 0.04$。

**为什么 α 这么小？** 因为图像 token 数量远大于动作 token（256 或 1024 个 vs 7 个），如果不降低世界模型损失的权重，图像损失会主导训练，淹没动作损失。

**世界模型为什么能帮助动作生成？** 三个原因：
1. 世界模型通过预测未来画面学到了环境物理规律，这种理解对操控任务至关重要；
2. 世界模型赋予系统"提前模拟"的能力，可以预判某个动作的后果；
3. 世界模型需要精确理解动作输入，这反过来帮助动作模型生成更合理的动作。

**Backbone 选择：** 以 Chameleon 作为初始化模型，因为它本身就是一个统一理解和生成的多模态模型，天然适合 WorldVLA 的框架。

## 四、实验与结果

### 4.1 实验设置

- **数据集**：LIBERO 基准，包括 LIBERO-Spatial（空间关系）、LIBERO-Object（物体识别）、LIBERO-Goal（目标泛化）、LIBERO-Long（长时域任务，10 个任务）和 LIBERO-90（90 个短时域任务用于预训练）。
- **评估指标**：动作模型用成功率（SR，Success Rate），每个任务跑 50 次取成功率；世界模型用 FVD（Fréchet Video Distance，越低越好）、PSNR、SSIM、LPIPS。
- **对比基线**：
  - 连续动作模型：Diffusion Policy、Octo、DiT Policy、Seer、OpenVLA-OFT、UVA
  - 离散动作模型：OpenVLA

### 4.2 主要结果

**与基线对比（Table 2）：**

| 模型 | 类型 | 预训练 | Spatial | Object | Goal | Long | 平均 |
|---|---|---|---|---|---|---|---|
| Diffusion Policy | 连续 | ✗ | 78.3 | 92.5 | 68.3 | 50.5 | 72.4 |
| OpenVLA | 离散 | ✓ | 84.7 | 88.4 | 79.2 | 53.7 | 76.5 |
| **WorldVLA (512×512)** | **离散** | **✗** | **87.6** | **96.2** | **83.4** | **60.0** | **81.8** |
| OpenVLA-OFT | 连续 | ✓ | 96.9 | 98.1 | 95.5 | 91.1 | 95.4 |

关键发现：
- WorldVLA 在**不使用大规模预训练**的情况下，比使用了预训练的 OpenVLA 高出 **5.3%** 的平均成功率（81.8% vs 76.5%），证明了统一框架的有效性。
- 512×512 分辨率比 256×256 高出约 2.7%，因为 Chameleon 的图像模块在 512×512 上预训练，且高分辨率提供了更精细的视觉信息，对精密抓取至关重要。

**世界模型帮助动作模型（Table 3 消融）：**

| 配置 | 动作模型 | 世界模型 | 动作块 | 本文掩码 | 平均 SR |
|---|---|---|---|---|---|
| 1 | ✓ | ✗ | ✗ | ✗ | 62.8% |
| 2 | ✓ | ✓ | ✗ | ✗ | **67.2%** |
| 4 | ✓ | ✗ | ✓ | ✓ | 76.6% |
| 5 | ✓ | ✓ | ✓ | ✓ | **78.1%** |

加入世界模型后，成功率提升约 **1.5%~4.4%**。可视化结果更直观：纯动作模型会在没抓住物体的情况下就移向目标位置，而 WorldVLA 会反复尝试抓取直到成功。

**动作模型帮助世界模型（Table 4）：**

| 模型 | 10 帧 FVD↓ | 50 帧 FVD↓ |
|---|---|---|
| 纯世界模型 | 250.0 | 718.6 |
| 动作世界模型 | 255.1 | **674.1** |

在长序列视频生成上，统一模型的 FVD 降低了 **44.5**（约 6.2% 的改善），说明动作理解能力确实增强了视觉生成质量。

### 4.3 消融实验

**动作注意力掩码的效果：**

这是最核心的消融。Table 3 中 row 3 vs row 4 的对比极为显著：
- 不用本文掩码，加入动作块生成后成功率从 62.8% **暴跌到 54.0%**（下降 8.8%）
- 使用本文掩码，成功率反而**飙升到 76.6%**（比不用动作块还高 13.8%）

这说明：(1) 自回归方式下朴素地生成动作块确实会导致严重的误差累积；(2) 提出的注意力掩码完美地解决了这个问题。

**世界模型 vs 视频预测模型：** 世界模型（以动作为条件生成未来帧）在所有任务上都帮助了动作模型，而视频预测模型（仅以任务描述为条件）在某些任务上反而有害。原因是不给动作条件会导致未来帧的歧义——同一个初始帧可以对应多个合理的未来状态。

**历史帧数（Table 5）：** 2 帧输入在成功率和推理速度之间取得最优平衡。1 帧不够（信息不足），4 帧带来的提升可忽略但速度下降明显。

**世界模型预训练（Table 6）：** 先用世界模型数据预训练、再微调动作模型，平均成功率从 62.8% 提升到 66.8%（+4.0%），说明世界模型学到的物理理解可以作为有效的初始化知识。

## 五、局限性与展望

- **离散图像 Tokenizer 的表达力有限**：当前使用的 VQ-GAN 在语义理解上弱于 CLIP 等视觉编码器。设计一个同时兼顾理解和生成的统一 Tokenizer 是重要的改进方向。
- **数据和模型规模有待扩展**：当前实验仅在 LIBERO 基准上验证，模型规模和训练数据量的扩展是提升性能的关键途径。
- **可引入辅助动作头**：在自回归生成之外，额外加一个动作头或许可以进一步提高抓取精度。
- **与连续动作模型仍有差距**：相比使用预训练的连续动作模型（如 OpenVLA-OFT 的 95.4%），WorldVLA 的 81.8% 还有不小的差距，离散化本身带来的信息损失是主要瓶颈。

## 六、灵魂三问

1. **它解决了什么问题？** 之前动作模型只会"做"不会"想"，世界模型只会"想"不会"做"——WorldVLA 把两者合二为一，让模型既能生成动作又能预测未来画面，互相增强。

2. **为什么这么做？** 最关键的设计选择是用**统一的离散 token 词表**把图像、文本、动作三种模态塞进同一个自回归 LLM。之所以不用扩散模型分别处理（像 UVA 那样），是因为离散自回归方案天然能复用大语言模型的强大预训练权重（Chameleon），且架构更简洁——一个模型、一套参数、两种用途。

3. **什么证据最有说服力？** Table 3 的 row 3 vs row 4 的对比最关键——仅仅换了注意力掩码（其他完全不变），成功率从 54.0% 跳到 76.6%，提升 22.6 个百分点。这不仅证明了掩码策略的有效性，更揭示了一个重要但之前被忽视的现象：自回归模型在动作域的泛化能力远弱于文本域，朴素的因果注意力对动作块生成是有害的。

## 七、个人总结

1. WorldVLA 的核心 idea 是将动作模型和世界模型统一到一个自回归 LLM 中，通过共享表征让"做动作"和"理解物理"互相增益。这个方向非常直觉——人在操作物体时也是同时在"想象后果"和"执行动作"。

2. 最大的优势是简洁优雅的统一框架和动作注意力掩码这个巧妙的技术贡献；最大的不足是离散 tokenization 带来的信息损失导致与连续动作模型（如 OpenVLA-OFT）仍有显著差距，同时实验仅限于 LIBERO 这一个基准，泛化能力尚未得到充分验证。

3. 对后续研究的启发：(1) 自回归模型在非预训练模态上的误差累积是一个普遍问题，注意力掩码设计值得在其他多模态生成任务中探索；(2) 如何设计一个同时擅长理解和生成的视觉 Tokenizer 仍是关键瓶颈；(3) 世界模型作为动作模型的"物理知识先验"这一范式可能比单纯扩大动作数据更高效。
