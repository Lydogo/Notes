# π0.7：一个可操控的通用机器人基础模型及其涌现能力

> 原标题：π0.7: a Steerable Generalist Robotic Foundation Model with Emergent Capabilities
> 作者：Physical Intelligence 团队（Bo Ai, Kevin Black, Chelsea Finn, Sergey Levine 等）
> 发表：2026
> 链接：https://pi.website/pi07

---

## 一、研究背景与动机

- 大语言模型展现出强大的组合泛化（Compositional Generalization）能力——能将训练数据中的知识以新方式组合来解决新问题。但在机器人领域，现有的视觉-语言-动作模型（VLA）远未达到这种能力，往往无法流畅执行所有训练过的指令，更难泛化到新任务
- 先前的机器人基础模型（如 π0、π0.5、π0.6）通常只使用简短的语言指令作为上下文（context），限制了模型对多样化数据的利用能力。当训练数据包含不同质量、不同策略的轨迹时，简单的训练方式会导致模型在不同模态间"平均化"，产生次优行为
- 本文的目标是：通过丰富的多模态上下文条件（Diverse Context Conditioning），让一个通用模型能利用大规模、混合质量的数据，并在测试时通过 prompt 精确操控模型行为，实现开箱即用的高性能表现和组合泛化

传统 VLA 的三个致命痛点正是 steerable 设计的出发点：
1. **只能响应"做什么"，无法控制"怎么做"**：给传统 VLA 一句"叠 T 恤"，它只能输出训练数据中最常见的平均化叠法，无法指定速度、质量、策略
2. **混合质量数据导致平庸行为**：包含失败案例和次优数据的训练集会让模型平均化不同策略，输出"不出错但也不优秀"的动作
3. **泛化能力无法被定向激活**：训练中未见的任务无法通过简单指令完成，模型只能被动复现固定任务

## 二、核心贡献

1. **多模态丰富 Prompt 设计**：除语言指令外，引入子任务指令（Subtask Instructions）、子目标图像（Subgoal Images）、Episode 元数据（速度/质量/错误标签）和控制模式标识，使模型能消歧大规模混合质量数据
2. **有效利用混合质量数据**：通过元数据标注，π0.7 能从低质量数据、失败数据、自主 rollout 数据中学习，甚至将 RL 训练的专家策略蒸馏回通用模型，数据越多性能越好
3. **涌现的组合泛化能力**：π0.7 展现出跨构型零样本迁移（Cross-Embodiment Transfer）、复杂指令跟随、通过语言 coaching 执行全新任务等涌现能力
4. **开箱即用匹配专家水平**：在多项灵巧操作任务上，单一通用模型无需任何任务特定微调，即可匹配甚至超越 RL 微调的专家策略

## 三、方法原理

### 3.1 整体框架

π0.7 是一个约 5B 参数的 VLA 模型，由三部分组成：
- **VLM 骨干**：基于 Gemma3 4B 初始化（含 400M 视觉编码器），处理图像、语言和各种 prompt 信息
- **MEM 视频历史编码器**：对多视角（最多 4 个摄像头）、多帧历史（最多 6 帧）进行时空压缩，输出固定数量的 token
- **动作专家（Action Expert）**：860M 参数的 Transformer，使用 Flow Matching 目标预测连续动作

核心训练目标：

$$\max_{\theta} \mathbb{E}_{\mathcal{D}} [\log \pi_\theta(a_{t:t+H} | o_{t-T:t}, C_t)]$$

其中 $C_t$ 是丰富的多模态上下文，包含语言指令、子任务、子目标图像、元数据等。

### 3.2 关键技术细节

#### Prompt 的多模态组成

| 组件 | 内容 | 训练 Dropout | 推理用法 |
|---|---|---|---|
| 子任务指令 $\hat{\ell}_t$ | 当前语义子任务的文本描述（如"打开冰箱门"） | 30%（有 subgoal 时） | 高层策略生成或人类 coaching |
| 子目标图像 $g_t$ | 多视角的近未来期望状态图像 | 75%（仅 25% batch 含 subgoal） | 轻量级世界模型生成 |
| Episode 元数据 $m$ | 速度（离散化步数）、质量（1-5 分）、错误标签 | 整体 15% + 各组件 5% | 设为最快速度/最高质量/无错误 |
| 控制模式 $c$ | joint 或 end-effector | 不 dropout | 按任务选择 |

训练时每个组件随机 dropout，使模型在推理时可灵活使用任意子集。

#### 子目标图像生成（世界模型）

子目标图像由一个基于 BAGEL（14B 混合 Transformer 图像生成/编辑模型）初始化的轻量级世界模型生成：

$$\max_{\psi} \mathbb{E}_{\mathcal{D}_g} [\mathcal{L}_{CFM}(g^*_t, g_\psi(o_t, \hat{\ell}_t, m))]$$

世界模型利用 web 数据预训练带来的语义和物理先验，能为新场景生成合理的子目标图像，有效弥补语言指令难以描述的视觉细节。推理时每 4 秒或子任务切换时异步刷新子目标。

#### Episode 元数据的作用机制——Steerable 的核心抓手

π0.7 大量使用次优数据（失败 demo、自主 rollout、RL 训练数据）。元数据让模型学会区分不同质量和策略：
- 训练时提供真实的速度/质量/错误标签
- 推理时设为"最高质量/最快速度/无错误"来引导模型输出最优行为
- 等效于一种对数据质量的条件建模，避免了简单平均不同模态的问题

元数据各维度的 steer 作用：

| 元数据维度 | 训练时 | 推理时 steer 方式 | steer 效果 |
|---|---|---|---|
| Overall Speed（步数） | 标注真实 episode 长度，离散化到 500 步区间 | 设为该任务第 15 百分位（最快） | 引导模型输出更高效、更少犹豫的动作 |
| Overall Quality（1-5 分） | 人工标注执行质量 | 设为 5（最高质量） | 引导模型输出最精准、最流畅的策略 |
| Mistake（错误标签） | 人工标注片段内是否有抓取失败等错误 | 设为 false | 引导模型避开错误行为模式 |
| Control Mode | 标注 joint 或 end-effector | 按任务需求选择 | 引导模型切换底层控制空间 |

**关键区别**：这里的 episode metadata 是 π0.7 独创的**语义级行为描述标签**，不同于 LeRobot 等框架中的 meta 数据。LeRobot 的 meta 是数据管理层面的结构化信息（fps、电机名称、数据维度等），用于数据解析和硬件适配；π0.7 的 metadata 则是对轨迹执行质量和策略的语义标注，直接以文本形式拼入 prompt 参与模型训练，是模型"可被引导"的条件信号。两者层次不同、用途不同，互不替代。

#### 知识隔离（Knowledge Insulation, KI）

VLM 骨干使用 FAST token 的离散交叉熵损失监督，动作专家可注意力访问 VLM 的全部激活，但**动作专家的梯度不回传到 VLM 骨干**。这使得 VLM 通过更稳定的离散损失训练，避免连续动作损失对视觉-语言表征的干扰。

#### Attention 模式

- 多视角图像 token 之间：双向注意力
- 子目标图像 token：block-causal，可看观测但后续文本不能看前面图像
- 文本 token：causal 注意力
- 动作 token：双向互看，可注意力访问 VLM 骨干所有激活

推理时支持分类器无关引导（Classifier-Free Guidance, CFG），通过构造 attention tree 将正/负样本打包到同一序列高效推理。CFG 公式：

$$\nabla_a \log \pi_\theta(a|o_t, C_t) + \beta \left(\nabla_a \log \pi_\theta(a|o_t, C_t) - \nabla_a \log \pi_\theta(a|o_t, C_t^{\text{uncond}})\right)$$

其中 $\beta \in \{1.3, 1.7, 2.2\}$，主要对 episode 元数据做 CFG，在灵巧任务上进一步放大 steer 效果。

### 3.3 Steerable 的全维度拆解

论文标题中的 steerable 不是单一概念，而是贯穿训练到推理的完整可引导体系，分为 5 个递进层次：

**第 1 层：任务目标的语义级 steer——"做什么"**

对传统 VLA 语言指令的升级。除顶层任务指令（"清理厨房"）外，支持子任务级指令（"先打开冰箱门"）进行 step-by-step coaching。关键能力：能精准响应反数据偏见的指令（如 Reverse Bussing：把盘子放垃圾桶、垃圾放餐盘框）和复杂空间指代（"拿起最大盘子上的水果"），模型遵循指令语义而非训练数据的平均行为。

**第 2 层：执行策略的精细化 steer——"怎么做"**

论文最核心的创新，通过 episode metadata 对执行速度、质量、错误倾向进行颗粒度极细的调控。训练时标注真实值，推理时设为最优值，让同一个模型能输出截然不同的行为策略。消融实验直接验证：去掉元数据后性能暴跌；加入元数据后，低质量数据反而能持续提升性能。

**第 3 层：空间动作的视觉级 steer——"做成什么样"**

通过子目标图像解决语言无法描述的细节。跨构型迁移中尤为关键：源机器人叠衣服用倾斜抓取，UR5e 更适合垂直抓取，子目标图像能引导模型放弃源策略、生成适配目标机器人的新动作。世界模型生成的子目标提供了"语言 + 视觉"的双维度 steer。

**第 4 层：跨域能力的定向 steer——"在新环境里怎么做"**

通过多模态 prompt 组合定向激活涌现的泛化能力：零样本跨构型迁移（BiPi → UR5e 叠衣服，成功率匹配人类专家）、技能组合式泛化（分步 coaching 完成从未见过的空气炸锅任务）。模型不是简单复制源行为，而是自主发现适合目标构型的全新操控策略。

**第 5 层：训练时的 steerability 底层设计**

steerable 不是推理时 prompt 工程调出来的，而是训练时专门设计的：
- **随机 Dropout**：训练时随机丢弃 prompt 各组件（subgoal 75%、metadata 15%、子任务 30%），让模型学会适配任意 prompt 子集
- **混合数据解耦**：元数据标签让不同质量/策略的数据不被平均化，模型学会"不同 prompt 对应不同输出"
- **块因果注意力**：专门设计的 attention mask 保证多模态 steer 信号能被完整接收且互不干扰

**具象示例（叠 T 恤任务）**：

| Steer 维度 | 不 steer 时的行为 | Steer 后的行为 |
|---|---|---|
| 语言指令 | 只能说"叠 T 恤"，模型选择默认叠法 | 可以说"先对折再叠袖子"，指定具体叠法 |
| 元数据 | 输出训练数据中的平均速度和质量 | 设 Speed=最快、Quality=5，输出最高效精准的策略 |
| 子目标图像 | 模型自行决定叠成什么样 | 提供目标图像，精确指定最终折叠状态 |
| 跨构型 | 在新机器人上复制源机器人的动作（可能失败） | 自主发现适配新机器人的抓取角度和策略 |

### 3.4 训练与优化

- **训练数据**：多平台（双臂移动/静态/单臂）多环境的遥操作数据、自主 rollout 数据（含 RL 训练数据和失败）、开源机器人数据集、第一人称人类视频、网络多模态数据
- **动作块（Action Chunk）**：50 步，使用 Flow Matching 生成；执行时取前 15 或 25 步
- **实时动作分块（RTC）**：训练时模拟 0-12 步推理延迟（50Hz 下最大 240ms），生成平滑动作轨迹
- **推理速度**：最小配置 38ms/chunk（单 H100），带 MEM 和 subgoal 最大 127ms；世界模型用 4×H100 张量并行 + 8bit 量化，25 步去噪生成 subgoal 仅需 1.25 秒

## 四、实验与结果

### 4.1 实验设置

- **机器人平台**：双臂移动操作器（6DoF×2）、静态双臂（BiPi）、UR5e 双臂 + Robotiq 夹爪、单臂系统
- **评估维度**：开箱即用灵巧性、指令跟随、跨构型迁移、组合任务泛化
- **对比基线**：π0.5、π0.6、π*0.6（RL 专家）、SFT 专家策略、人类遥操作

### 4.2 主要结果

**开箱即用的灵巧操作**：
- 在咖啡机操作、折叠衣物、搭建盒子等 π*0.6 任务上，单一 π0.7 模型的成功率与任务特定 RL 专家相当
- 在折衣和搭盒子任务上，π0.7 的吞吐量（successes/hour）甚至超过 RL 专家
- 在切西葫芦、削皮、换垃圾袋等额外灵巧任务上，同样匹配 SFT 专家

**指令跟随**：
- 在 4 个未见厨房 + 2 个未见卧室环境中，π0.7 的指令跟随成功率显著优于 π0.5 和 π0.6
- 对复杂引用指令（如"拿起最大盘子上的水果"），π0.7 + subgoal 达到最高成功率
- 在"反向"任务（与数据集偏差对抗）中，π0.7 能成功打破数据偏差

**跨构型迁移**：
- 简单重排任务：π0.5/0.6/0.7 均可跨构型迁移
- 大形态差异任务（BiPi → UR5e 衬衫装袋）：π0.7 显著优于先前模型
- 灵巧任务（折叠 T 恤从 BiPi → UR5e，零样本）：π0.7 (GC) 达到 85.6% 任务进度、80% 成功率，**匹配顶级人类遥操作员的零样本表现**（90.9% / 80.6%）
- 模型会自主发现适合目标构型的新操控策略（如单臂 pick-and-place 替代双臂协作）

**组合任务泛化**：
- 短 horizon 新任务（压法压壶、舀米饭、擦拭办公用品等）：直接 zero-shot 成功
- 长 horizon 新任务（操作空气炸锅、烤面包等）：通过人类语言 coaching 分步引导完成，然后用 coaching 数据训练高层策略实现全自主

### 4.3 消融实验

| 消融条件 | 关键结论 |
|---|---|
| 去除 episode 元数据 | 所有任务吞吐量显著下降，模型无法消歧混合质量数据 |
| 去除 eval/自主数据 | 性能下降，说明次优数据（配合元数据）对性能有正贡献 |
| 不同质量数据比例 × 有/无元数据 | 无元数据时加入更多混合质量数据会损害性能；有元数据时持续提升 |
| 去除最高任务多样性的 20% 数据 | 组合泛化性能显著下降（对比随机去除 20%），说明任务多样性驱动泛化 |

## 五、局限性与展望

- 零样本泛化成功率（60-80%）仍低于见过的任务（>90%），仍有提升空间
- 在如此大规模数据集中，严格定义"未见任务"十分困难——模型可能通过"重混"已见技能实现所谓的泛化，但作者认为这正是组合泛化的本质
- 未来方向：利用 π0.7 的高可操控性，在测试任务上高效学习（如更详细的语言 coaching 或自主 RL）

## 六、灵魂三问

1. **它解决了什么问题？** 之前的 VLA 模型无法有效利用混合质量数据、难以零样本泛化到新任务和新机器人构型；π0.7 通过丰富的多模态 prompt 条件化，使一个通用模型无需微调即可匹配专家水平，并展现跨构型迁移和组合泛化的涌现能力。

2. **为什么这么做？** 核心设计选择是用 episode 元数据（质量/速度/错误标签）来条件化训练。不用这种方案（即直接混合训练），模型会"平均化"不同质量的行为模式，导致加入更多数据反而性能下降。元数据条件化让模型学会区分好坏策略，推理时只需 prompt 为最高质量即可。这比传统的数据过滤方案（费力且丢弃信息）高效得多。

3. **什么证据最有说服力？** 最有力的是 Fig. 18 左图的 scaling 消融：无元数据时，加入更多混合质量数据会让性能下降；有元数据时则持续提升。这直接证明了元数据条件化不仅解决了数据质量问题，还解锁了数据规模的 scaling 效应。另外，跨构型折衣实验中 π0.7 匹配人类顶级操作员的零样本表现（80% vs 80.6% 成功率），有力说明了组合泛化的涌现。

## 七、个人总结

1. π0.7 的核心 idea 是"通过丰富的多模态 prompt 来解锁大规模混合质量数据的价值"——与其费力过滤数据，不如给数据打上质量标签让模型自己学会区分。Steerable 是这篇论文的灵魂：传统 VLA 只能让机器人"完成任务"，而 π0.7 的 steerable 设计能精准决定机器人"用什么方式、什么质量、什么策略、什么细节"完成任务。
2. 最大优势是实现了真正意义上的"一个模型搞定一切"——开箱即用匹配 RL 专家、跨构型零样本迁移、语言 coaching 学新任务，且三者共存于同一模型。不足在于零样本泛化成功率仍有差距，且世界模型推理开销较大（14B 模型 + 4×H100）。
3. 对后续研究的启发：a) 数据质量标注 + 条件化训练是比数据过滤更可扩展的范式；b) 语言 coaching → 高层策略的 pipeline 提供了一种几乎零成本的新任务部署方式；c) 跨构型迁移中模型自主发现新策略的现象暗示大规模 VLA 可能已经具备某种程度的"物理推理"能力。

## 八、Metadata 与 LeRobot 框架的关系

### 概念辨析

π0.7 的 episode metadata 和 LeRobot 的 meta 数据**不是同一种数据**，两者在不同层次服务于不同目的：

| 对比维度 | π0.7 Episode Metadata | LeRobot Meta 数据 |
|---|---|---|
| **本质** | 语义级行为描述标签 | 数据管理层结构化信息 |
| **内容** | 速度、质量评分、错误标签、控制模式 | fps、电机名称、数据维度、摄像头配置等 |
| **用途** | 拼入 prompt 文本，作为模型的条件信号参与训练和推理 | 数据解析、硬件参数匹配、数据加载 |
| **来源** | 人工标注 + 自动统计（episode 步数） | 数据采集时自动生成 |
| **模型是否看到** | 是，以文本形式输入模型 | 否，仅供数据 pipeline 使用 |

π0.7 的 metadata 是独立创造的概念——它借鉴了扩散模型中 prompt expansion 的思路，将数据质量信息编码为条件信号，是一种模型能理解的语义标注，而非数据格式描述。

### 基于 LeRobot 复现时的处理

如果想基于 LeRobot 框架复现 π0.7 的 metadata 机制，需要在 LeRobot 原生 meta 之上新增一层：

1. **扩展 episode meta 字段**：在数据集配置中新增 `episode_speed`（int，离散化步数）、`episode_quality`（int，1-5）、`has_mistake`（bool）、`control_mode`（str，joint/ee）四个字段
2. **数据预处理标注**：给每条轨迹打上对应的 metadata 标签，与 LeRobot 原生 meta 一起存储（两者互不冲突）
3. **训练时拼入 prompt**：从 episode meta 中提取这 4 个字段，格式化为文本（如 `Speed: 2000. Quality: 5. Mistake: false. Control Mode: joint.`），与语言指令、子目标图像拼接成完整 prompt 输入模型。LeRobot 原生 meta 仍仅用于数据解析和硬件参数匹配，不进入模型
