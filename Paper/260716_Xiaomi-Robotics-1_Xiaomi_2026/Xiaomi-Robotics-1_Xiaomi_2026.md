## 一、研究背景与动机

这篇论文的核心问题是：机器人领域能否像大语言模型和视觉语言模型一样，通过扩大数据和模型规模获得稳定提升。现有视觉-语言-动作模型（Vision-Language-Action, VLA）已经显示出泛化潜力，但机器人数据的获取方式仍主要依赖真实机器人遥操作，成本高、速度慢、受硬件绑定，并且数据集中在较窄的任务和环境上。

Xiaomi-Robotics-1 的切入点是把可扩展的 UMI 手持夹爪轨迹作为大规模预训练数据来源，再用较小但高质量的跨本体机器人数据做后训练对齐。它不是只提出一个新架构，而是系统性回答两个 scaling 问题：更大的真实操作数据是否降低动作误差，以及更强的预训练模型是否能迁移到真实机器人 out-of-the-box 表现。

## 二、核心贡献

1. **提出大规模 VLA 训练配方**：使用超过 100K 小时真实世界 UMI 操作轨迹进行预训练，再用约 10K 小时跨本体数据进行后训练，把状态转移理解对齐到机器人本体和命令式任务指令。
2. **构建自动语言标注流水线**：将长轨迹切成固定长度片段，用 Qwen3.5-27B 标注夹爪与物体的状态转移描述，使超大规模轨迹能获得细粒度语言条件。
3. **采用 Qwen3-VL + DiT 的 MoT 架构**：VLM 编码观察和语言，DiT 用 flow matching 生成动作 chunk；同时给 VLM 加 Choice Policies 辅助动作监督以加速收敛。
4. **给出较完整的 scaling 证据**：数据规模、模型规模、真实机器人 out-of-the-box、少样本微调和四个仿真 benchmark 都显示提升趋势。

## 三、方法原理

### 3.1 整体框架

Xiaomi-Robotics-1 是端到端 VLA 策略。输入为当前视觉观察、语言指令和机器人状态，输出未来一段动作 chunk。训练分两阶段：

| 阶段 | 数据 | 语言形式 | 目标 |
| --- | --- | --- | --- |
| 预训练 | 超过 100K 小时 UMI 真实轨迹 | 状态转移描述 | 学会把当前场景变到语言描述的目标状态 |
| 后训练 | 约 10K 小时跨本体轨迹 | 人类命令式指令 | 对齐到机器人本体、动作空间和真实任务提示 |

这种设计把“可扩展数据来源”和“机器人可执行动作”拆开：UMI 数据负责规模和多样性，机器人数据负责本体对齐。

### 3.2 关键技术细节

模型采用 Mixture-of-Transformers（MoT）结构，由 Qwen3-VL 作为 VLM backbone，配一个 Diffusion Transformer（DiT）动作生成器。论文给出三种规模：

| 模型 | VLM 参数 | DiT 参数 | 总参数 |
| --- | ---: | ---: | ---: |
| Xiaomi-Robotics-1-2B | 2.1B | 470M | 2.6B |
| Xiaomi-Robotics-1-5B | 4.4B | 604M | 5.1B |
| Xiaomi-Robotics-1-10B | 8.8B | 1.5B | 10.5B |

DiT 通过 flow matching 预测动作 chunk，从噪声动作逐步积分到干净动作。推理时使用 5 步 Euler integration，步长为 0.2。直观上，它把动作生成看成从随机动作到真实动作的连续流场建模，比离散动作 token 更适合连续控制。

另一个关键点是 **VLM 辅助动作监督**。作者用 Choice Policies 让 VLM 直接预测多个候选动作 chunk 和对应分数，再用 winner-takes-all 的 L1 loss 训练最接近真值的候选。这个辅助任务让 VLM 表征更偏向动作生成。但 DiT 不能注意到这些 action-related tokens，否则会出现“复制 VLM 动作”的捷径，因此 DiT 只看视觉和语言 token 的 KV cache。

### 3.3 训练与优化

预训练数据来自 UMI 手持夹爪和第一视角相机，覆盖家庭、商业、工业、办公室、户外等场景。自动标注流水线把轨迹切成等长片段，用 Qwen3.5-27B 描述夹爪和物体的状态变化。作者称 100K 小时数据约两周完成标注。

训练目标为：

```text
L = LFlow + LRegression + 0.1 * LNTP
```

其中 `LFlow` 训练 DiT 动作流，`LRegression` 训练 VLM Choice Policies，`LNTP` 保留 VLM 的视觉语言能力。预训练中视觉语言数据与 UMI 轨迹按 1:9 采样。

后训练使用超过 7,200 小时自采移动操作/双臂机器人数据、超过 1,000 小时指令标注 UMI 数据，以及 Bridge V2、RT-1、DROID 等开源机器人数据，总计约 10K 小时。动作空间采用相对末端执行器位姿、移动底盘速度和腰部相对位置，并对不同本体缺失维度做 loss mask。

## 四、实验与结果

### 4.1 实验设置

实验围绕四个问题展开：预训练数据和模型是否可 scaling，预训练能力是否能迁移到真实机器人后训练，是否能用少量数据适配新任务，以及在仿真 benchmark 上是否优于现有基座策略。

真实机器人 out-of-the-box 评估包含 4 个训练中见过任务类型但环境/物体未见过的任务：shoe storage、bag packing、table organization、sofa tidying。少样本微调评估包含 4 个完全新任务：phone packing、laundry loading、printer refilling、box packing。

### 4.2 主要结果

**预训练 scaling**：在 20K 小时 UMI 子集上，5B 模型随数据从 12.5%、25%、50%、100% 增加，验证动作 MSE 持续降低。2B、5B、10B 模型也随模型规模扩大降低动作误差，但作者指出数据规模带来的差异更明显。

**后训练 out-of-the-box**：

| 对比维度 | 结果 |
| --- | --- |
| 无动作预训练 | 平均成功率 26% |
| 12.5% 预训练数据 | 平均成功率 53% |
| 100% 预训练数据 | 平均成功率 75% |
| 2B / 5B / 10B | 平均成功率 61% / 75% / 79% |

最有说服力的是 shoe tidying：无动作预训练完全失败，而 100% 预训练数据模型达到 75%；10B 在该任务上达到 92%。这说明预训练对接触丰富、需要细粒度动作的任务贡献更大。

**少样本微调**：

| 设置 | Xiaomi-Robotics-1 | π0.5 |
| --- | ---: | ---: |
| 低数据：四任务共 36 小时，平均每任务 <10 小时 | 75% 成功率，90% progress | 40% 成功率，66% progress |

在 printer refilling 这类变形物体任务上，Xiaomi-Robotics-1 低数据设置把最好 baseline 的成功率从 20% 提到 70%。在 laundry loading 长程移动操作中，低数据设置达到 80% 成功率和 96% progress。

**仿真 benchmark**：

| Benchmark | Xiaomi-Robotics-1 结果 | 关键信息 |
| --- | ---: | --- |
| RoboCasa | 74.5% avg success | 高于 World2Act 72.6% |
| RoboCasa365 | 57.4% avg success | 高于 ABot-M0.6 46.6%；Composite-Unseen 为 32.1% |
| VLABench | 59.1 SR / 70.3 PS / 69.9 IS | SR 和 PS 最高，IS 接近 ERVLA |
| RoboDojo | 20.07 avg score / 13.93% SR | 平均 score 明显高于 Hy-Embodied-0.5-VLA 的 13.07 |

### 4.3 消融实验

论文中最重要的消融不是单个模块删除，而是 scaling 曲线：

| 消融对象 | 观察 |
| --- | --- |
| 预训练数据量 | 数据越多，验证动作误差越低；低数据比例会更快过拟合 |
| 模型规模 | 2B -> 5B -> 10B 后训练真实机器人成功率单调上升 |
| 动作预训练有无 | 无动作预训练的后训练模型平均 26%，12.5% 数据已提升到 53% |

这说明后训练不是从零学机器人技能，而是在调用和对齐预训练得到的动作生成能力。

## 五、局限性与展望

作者主要强调 scaling 的正向趋势，但仍有几个部署层面的限制：

1. **数据规模强但可复现性有限**：100K 小时 UMI 和 10K 小时跨本体数据不是普通实验室能直接复制的资源。
2. **本体对齐仍依赖后训练数据**：UMI 预训练不能直接解决机器人形态、动作维度和控制约束差异；移动操作、双臂和静态机械臂仍需要大量对齐数据。
3. **历史记忆能力不足**：RoboDojo 的 memory 维度低于显式建模记忆的 Hy-Embodied-0.5-VLA，说明当前模型更像强单步/短程策略，而不是长程记忆策略。
4. **开放世界失败模式未充分展开**：论文展示 suitcase packing 等长程任务，但未系统报告真实家庭中的失败类别、恢复策略、延迟和安全约束。

后续值得关注的是：将大规模 UMI 预训练与显式记忆、在线纠错和跨机器人动作规范化结合，是否能进一步提升真实部署稳定性。

## 六、灵魂三问

1. **它解决了什么问题？**  
它解决的是 VLA 缺少可扩展真实操作数据和跨本体对齐配方的问题。相比只依赖机器人遥操作数据，它把 UMI 轨迹作为大规模预训练来源，再用机器人数据把能力转成可执行策略。

2. **为什么这么做？**  
因为直接收集 100K 小时机器人遥操作成本过高，而 UMI 设备能低成本进入更多环境。状态转移语言描述比人工命令标注更适合自动化预训练，后训练再把这种描述式条件转为人类实际会说的命令式指令。

3. **什么证据最有说服力？**  
最有说服力的是后训练 out-of-the-box 的 scaling 迁移：无动作预训练平均 26%，12.5% 预训练数据到 53%，完整预训练到 75%；这说明预训练不是只降低离线动作误差，而是真能转化为未见环境中的真实机器人成功率。

## 七、个人总结

1. Xiaomi-Robotics-1 的核心不是单纯换一个动作头，而是把“UMI 规模化预训练 + 跨本体后训练对齐”做成 VLA scaling pipeline。
2. 最大优势是证据链完整：离线误差、真实机器人、少样本微调和仿真 benchmark 都支持 scaling；最大弱点是依赖小米级别的数据和基础设施，普通研究组很难复现同等规模。
3. 对 VLA 方向的启发是：未来竞争可能不只在模型结构，而在可扩展数据采集、自动语言标注、动作空间统一和后训练 recipe 的系统工程能力。
