## 一、研究背景与动机

端到端机器人策略通常只看当前观察，或者把少量历史帧直接塞进模型上下文。但真实长程任务需要不同粒度的记忆：短期需要记住刚才的视觉细节、遮挡和失败尝试，长期只需要记住“已经完成了哪些步骤”“某个物品被放到了哪里”这类语义状态。

直接把十几分钟的高频视频历史输入 VLA 在计算上不可行，也会带来延迟问题。MEM 的核心动机是给 VLA 一个混合模态的记忆系统：用视频编码器保留短期视觉记忆，用语言摘要保留长期语义记忆，从而支持 15 分钟级别的真实机器人任务。

## 二、核心贡献

1. **提出 Multi-Scale Embodied Memory（MEM）**：把短期视频记忆和长期语言记忆组合进 VLA，使策略能处理长程任务、部分可观测和失败后的上下文适应。
2. **设计无新增参数的视频记忆编码器**：在 ViT 中交替加入空间注意力和因果时间注意力，压缩过去帧，只把当前时刻 token 传给 VLA backbone。
3. **引入语言记忆更新机制**：高层策略预测子任务指令和更新后的自然语言记忆，避免把所有历史子任务简单拼接造成 train-inference 分布偏移。
4. **在 π0.6 上验证真实机器人效果**：MEM 让 VLA 完成 recipe setup、clean kitchen、grilled cheese 等需要最长约 15 分钟记忆的任务，并在核心记忆能力任务上优于 Pool Memory 和 Proprio Memory。

## 三、方法原理

### 3.1 整体框架

MEM 把策略分解成高层和低层：

```text
高层策略：根据当前观察、任务目标、旧语言记忆，预测下一子任务和新语言记忆
低层策略：根据短期视频观察、子任务和任务目标，输出连续动作 chunk
```

直观上，高层负责“我现在做到了哪一步，接下来该做什么”，低层负责“在最近几秒的视觉上下文里怎么动手”。这比把所有历史帧直接输入模型更符合机器人任务的时间结构。

### 3.2 关键技术细节

**语言长期记忆** 是一个自然语言摘要 `m_t`。高层策略 `πHL` 读取当前观察、任务目标和旧记忆，输出下一子任务 `l_{t+1}` 与新记忆 `m_{t+1}`。训练数据由 LLM 根据机器人 episode 的子任务标注和成功/失败状态生成。关键不是记录所有事件，而是压缩掉未来不再需要的信息。

作者特别对比了 naive text memory：如果只是拼接所有历史子任务，推理时会遇到重复失败导致的分布偏移。例如训练示范中“拿起碗 -> 放入柜子”通常只出现一次，但推理中模型可能多次失败并重复同一子任务。MEM 的语言记忆在失败时可以选择不更新，从而减少这种偏移。

**视频短期记忆** 解决遮挡、动态理解和失败后策略调整。MEM 改造标准 ViT：普通层做空间注意力，每 4 层加入一次时间注意力，对同一空间 patch 的不同时间步做因果注意力。复杂度从朴素时空联合注意力的 `O(n^2 K^2)` 降到 `O(K n^2 + n K^2)`。

更重要的是，视频编码器不新增可学习参数，只改变注意力模式并加入固定的时间位置编码；当历史长度 `K=1` 时，它退化成原来的单帧 ViT。这使得它能从预训练 VLM 直接初始化，不破坏单图像能力。

### 3.3 训练与优化

作者把 MEM 集成进 π0.6。模型初始化自 Gemma3-4B VLM，动作部分同时使用 FAST 离散动作 token 预测和 860M 参数的 flow-matching action expert。输入分辨率为每路相机 448x448，最多使用 4 路相机。

预训练时使用 6 帧观察：5 个历史帧加当前帧，帧间隔为 1 秒。后训练时可把观察记忆扩展到更长范围，实验中最高到 18 帧、54 秒。语言记忆则负责跨分钟级别的语义状态。

对于机器人 proprioception，π0.6 原本用文本表示机器人状态；MEM 改用连续状态 embedding，把每个历史状态投影到 backbone embedding 空间，避免历史状态文本 token 爆炸。

## 四、实验与结果

### 4.1 实验设置

实验回答三个问题：

1. MEM 是否能完成需要最长 15 分钟记忆的真实任务。
2. 记忆是否能带来短程任务中的 in-context adaptation。
3. 与其他 VLA 记忆方法相比，MEM 是否更均衡。

长程任务包括 recipe setup 和 clean kitchen。分析任务包括三杯交换、找隐藏物体、从购物袋取完所有东西、咖啡精确舀两勺、grilled cheese、window cleaning 等。对照方法包括无记忆 π0.6、Pool Memory、Proprio Memory，以及只在后训练阶段加入 MEM 的消融版本。

### 4.2 主要结果

**长程任务**：Figure 6 显示，π0.6-MEM 在 recipe setup、clean kitchen 和平均 progress 上都明显高于无记忆、只有视频记忆、只有文本记忆和 naive text+video memory。图中平均 progress 约从无记忆的三成提升到 MEM 的七成左右；clean kitchen 上 MEM 接近九成 progress。

**短程适应**：在 pick up chopstick 和 open fridge 两个任务中，MEM 通过短期记忆记住刚才失败的策略并切换行为。图中明确标注：chopstick 任务成功率提升 11 个百分点，open fridge 提升 62 个百分点。后者尤其说明记忆不是只用于长程规划，也能用于失败后的策略修正。

**核心记忆能力对比**：

| 方法 | 能力特点 | 主要问题 |
| --- | --- | --- |
| No Memory | 在非记忆复杂操作上仍强 | 部分可观测、计数、时序任务接近随机 |
| Pool Memory | 压缩历史图像，简单任务有帮助 | 平均池化丢失时序和空间细节 |
| Proprio Memory | 适合记住机器人自身状态 | 对环境状态记忆弱 |
| MEM | 视频短期 + 语言长期，整体最好 | 系统复杂度更高，需要对应训练数据 |

Figure 8 中，MEM 是唯一在所有核心记忆任务上都表现较强的方法；尤其在 window cleaning、find object、unpack groceries 这类需要环境状态记忆的任务上优势明显。

**记忆预训练的重要性**：Figure 9 显示，只在后训练引入记忆明显弱于预训练阶段就学习记忆的 MEM。作者认为，机器人和非机器人视频混合预训练使模型学会如何使用历史帧，即使后训练时记忆 horizon 从 5 秒扩到接近 1 分钟也能受益。

### 4.3 消融实验

关键消融集中在三点：

| 消融 | 结论 |
| --- | --- |
| 去掉视频短期记忆 | 容易在擦拭、洗盘子等任务中不知道持续了多久，或者无法利用失败尝试进行调整 |
| 去掉语言长期记忆 | 难以记住 recipe 步骤、柜门是否关闭、哪些物品已处理 |
| 用 naive 语言历史替代压缩摘要 | 历史指令重复导致 train-inference 分布偏移，效果低于模型预测摘要 |

这说明 MEM 的有效性来自两种记忆的互补，而不是“多塞几帧”本身。

## 五、局限性与展望

1. **依赖高层子任务/记忆监督**：语言记忆训练需要从 episode 标注生成摘要，数据准备不如纯端到端动作学习简单。
2. **记忆仍在单 episode 内**：论文结论中也指出，未来要扩展到跨天、跨月的部署记忆和持续学习。
3. **显式高低层分解带来系统复杂度**：高层记忆更新错误会影响低层执行，低层失败又需要高层正确处理是否更新记忆。
4. **结果多为 progress 图，精确数值有限**：论文展示趋势清楚，但部分柱状图没有在正文中逐项给出数值，复现实验时需要读图或等待更完整表格。

MEM 的方向很重要：它把 VLA 从“反应式策略”推进到“有状态策略”。下一步更有价值的问题是记忆如何和安全恢复、任务重规划、在线学习结合，而不是单纯延长视频上下文。

## 六、灵魂三问

1. **它解决了什么问题？**  
它解决的是 VLA 缺乏长短期记忆的问题。传统 VLA 主要依赖当前观察，无法稳定处理部分可观测、计数、长程任务步骤和失败后的策略切换。

2. **为什么这么做？**  
因为机器人记忆天然是多尺度的：短期需要视频细节来处理遮挡和失败，长期只需要压缩语义状态。用视频记忆覆盖最近几十秒，用语言摘要覆盖十几分钟，比直接输入完整历史更有效率，也更符合任务结构。

3. **什么证据最有说服力？**  
最有说服力的是长程任务和 in-context adaptation 的组合证据：MEM 不仅让 recipe setup / clean kitchen 的 progress 大幅提升，还在 open fridge 中带来 62 个百分点提升，说明记忆同时改善长程语义状态和短程纠错。

## 七、个人总结

1. MEM 的核心可以概括为：用语言记住“发生过什么”，用视频记住“刚才怎么发生的”。
2. 最大优势是机制划分清晰、和真实机器人任务强相关；最大弱点是训练和系统集成成本高，不是一个轻量插件。
3. 对 VLA 研究的启发是，未来长程机器人不可能只靠更大单帧模型解决，必须显式处理记忆的压缩、更新和错误恢复。
