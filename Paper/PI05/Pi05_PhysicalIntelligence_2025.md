# π0.5：具备开放世界泛化能力的视觉-语言-动作模型

## 一、研究背景与动机

- **核心问题**：机器人要真正有用，必须在实验室之外的真实世界中完成实际任务。视觉-语言-动作模型（Vision-Language-Action, VLA）虽然在端到端机器人控制上已展示了令人印象深刻的结果，但其在开放世界中的泛化能力仍是一个悬而未决的问题。
- **现有方法的不足**：
  - 现有 VLA 模型通常只在与训练数据高度匹配的环境中进行评估，难以应对从未见过的新场景。
  - 简单地通过暴力扩展机器人数据采集来覆盖所有可能场景是不可行的，尤其对于"清洁厨房"这类复杂、长时间跨度的任务。
  - 虽然一些方法通过在更多环境中收集数据来提升简单技能（如抓取物体、开抽屉）的泛化性，但这对更复杂的长时间任务效果有限。
- **本文出发点**：类比人类从多种信息源（亲身经历、他人教导、阅读知识等）中获取经验并灵活应用，π0.5 旨在通过**异构数据联合训练**（co-training），将来自不同机器人平台、高层语义预测、网络数据和口头指令等多种知识源整合到一个统一的 VLA 框架中，实现面向新环境的广泛泛化。

## 二、核心贡献

1. **提出 π0.5 模型**：一个基于 π0 的 VLA 系统，通过异构多源数据联合训练实现广泛的开放世界泛化，能在**从未见过的真实家庭**中控制移动操作机器人执行复杂任务。
2. **设计层次化推理架构**：同一模型既作为高层策略（预测语义子任务，如"拿起盘子"），又作为低层策略（输出连续动作），通过类似思维链（Chain-of-Thought）的方式实现长时间跨度的复杂任务规划与执行。
3. **系统性验证联合训练配方**：通过详尽的消融实验，证明了跨具身（cross-embodiment）数据、网络数据、高层语义预测和口头指令等各数据源对泛化性能的关键作用。
4. **首次展示端到端学习系统在全新家庭中完成长时间、灵巧操作任务**（如清洁厨房或卧室，持续 10-15 分钟）。

## 三、方法原理

### 3.1 整体框架

π0.5 采用**两阶段训练**：

- **预训练阶段**（Pre-training）：在大规模异构数据混合上训练标准自回归 Transformer，所有任务（包括机器人动作）均用离散 token 表示（采用 FAST 动作分词器），实现简单、高效、可扩展的训练。
- **后训练阶段**（Post-training）：将模型特化为移动操作任务，加入**动作专家**（Action Expert），通过流匹配（Flow Matching）生成连续动作块，同时保留文本预测能力。

**推理时**：模型先根据高层指令（如"清洁厨房"）预测语义子任务（如"拿起盘子"），然后基于该子任务预测低层机器人动作。

### 3.2 关键技术细节

#### 模型架构

π0.5 基于 PaliGemma VLM（SigLIP 400M + Gemma 2.6B）构建，能同时输出文本 token 和连续动作：

- **输入**：多视角图像 $I_1^t, \ldots, I_n^t$、本体感受状态 $q_t$（关节角度、末端执行器位姿、底盘速度等）、任务指令 $\ell$
- **输出**：高层子任务预测 $\hat{\ell}$（文本 token）和低层动作块 $a_{t:t+H}$（连续值）

模型的联合分布被分解为：

$$
\pi_\theta(a_{t:t+H}, \hat{\ell} | o_t, \ell) = \pi_\theta(a_{t:t+H} | o_t, \hat{\ell}) \cdot \pi_\theta(\hat{\ell} | o_t, \ell)
$$

动作分布不依赖原始高层指令 $\ell$，只依赖预测出的子任务 $\hat{\ell}$，实现了高低层推理的解耦。

#### 离散与连续动作表示的结合

模型同时训练两种动作预测方式：
- **FAST 离散 token**：通过自回归采样预测动作（训练效率高）
- **流匹配连续动作**：通过迭代去噪生成连续动作（推理速度快、精度高）

联合损失函数为：

$$
\mathbb{E}_{D,\tau,\omega}\left[\mathcal{H}(x_{1:M}, f_\theta^\ell(o_t, \ell)) + \alpha \|\omega - a_{t:t+H} - f_\theta^a(a_{t:t+H}^{\tau,\omega}, o_t, \ell)\|^2\right]
$$

其中 $\mathcal{H}$ 为文本 token（包含 FAST 编码的动作 token）的交叉熵损失，第二项为流匹配损失，$\alpha$ 为权衡系数。这使得预训练阶段可以仅用离散 token 高效训练（$\alpha=0$），后训练阶段再引入动作专家进行连续动作生成。

#### 动作专家（Action Expert）

- 独立的 300M 参数 Transformer，通过自注意力与 VLM 主干交互
- 采用 Adaptive RMSNorm 注入流匹配时间步信息
- 推理时使用 10 步去噪生成动作，支持 50Hz 实时控制

#### 注意力掩码设计

- 图像、文本提示、本体感受 token 使用**双向注意力**
- FAST 动作 token 自回归地关注前缀和之前的动作 token
- 动作专家 token 关注前缀和彼此，但**不关注 FAST 动作 token**（避免两种动作表示之间的信息泄漏）
- 信息从 VLM 单向流向动作专家

### 3.3 训练与优化

#### 数据源

**预训练数据**（97.6% 不来自目标移动操作平台）：

| 数据源 | 缩写 | 描述 |
|--------|------|------|
| 多样移动操作 | MM | 约 400 小时，100+ 家庭环境中移动操作机器人数据 |
| 多环境非移动操作 | ME | 固定单臂/双臂机器人在多样家庭环境中采集 |
| 跨具身实验室数据 | CE | 实验室环境多种机器人多种任务，含开源 OXE 数据集 |
| 高层子任务预测 | HL | 将高层指令分解为子任务的标注数据 |
| 多模态网络数据 | WD | 图像描述、VQA、物体定位等网络视觉-语言数据 |

**后训练数据**额外引入：

| 数据源 | 缩写 | 描述 |
|--------|------|------|
| 口头指令 | VI | 专家用户通过语言实时"遥操作"机器人的高层示教数据 |

#### 训练超参数

- 预训练：280k 梯度步
- 后训练：80k 梯度步，流匹配权重 $\alpha = 10.0$
- 动作归一化至 $[-1, 1]$（使用各维度 1% 和 99% 分位数）
- 图像增强：随机裁剪（95%）、调整大小、旋转（±5°）、颜色抖动
- 流匹配时间步采样使用偏向低时间步的 Beta 分布：$p(\tau) = \text{Beta}(\frac{s-\tau}{s}; \alpha=1.5, \beta=1)$
- 动作块长度 $H=49$（共 50 步）

#### 机器人平台

- 两种移动操作机器人，均配备：2 个 6-DoF 手臂 + 平行夹爪、4 个 RGB 摄像头（前/后/双腕部）、全向轮底盘、躯干升降机构
- 状态/动作空间维度为 18-19 维
- 控制频率 50Hz，使用简单 PD 控制器跟踪目标位姿，无额外轨迹规划或碰撞检测

## 四、实验与结果

### 4.1 实验设置

- **评估环境**：所有实验均在训练中**从未见过**的环境中进行
  - 模拟家庭环境（mock homes）：用于可控的定量对比
  - 3 个真实家庭：用于最终真实场景评估
- **评估任务**：厨房清洁（餐具放入水槽、物品放入抽屉）和卧室清洁（衣物放入篮子、整理床铺），每个任务持续 2-15 分钟
- **评估指标**：任务进度百分比（基于详细的评分标准，如放入水槽的餐具数量）
- **对比基线**：π0、π0-FAST+Flow、不同数据混合消融版本、GPT-4 高层策略、人类高层策略（oracle）

### 4.2 主要结果

#### 真实家庭泛化（Q1）

π0.5 在 3 个全新真实家庭的厨房和卧室中成功执行了多种任务。以"餐具放入水槽"为例，在 Home 1-3 中，任务进度平均达到 **60%-80%** 以上。模型自主决定子任务顺序，完全端到端控制。

#### 泛化随场景数量的扩展（Q2）

| 训练场景数 | 平均任务进度 |
|-----------|------------|
| 3 个 | ~25% |
| 12 个 | ~35% |
| 53 个 | ~55% |
| 104 个 | ~65% |
| 含测试环境数据的对照 | ~65% |

关键发现：104 个场景训练的模型达到了与**在测试环境上直接训练**的模型相当的性能，证明联合训练配方有效弥合了泛化差距。

#### 联合训练配方消融（Q3）

| 消融条件 | 相对性能影响 |
|---------|------------|
| 完整 π0.5 | 基线（最佳） |
| 无 WD（网络数据） | 总体影响不大，但对 OOD 物体的语言跟随显著下降 |
| 无 ME（多环境数据） | 显著下降 |
| 无 CE（跨具身数据） | 显著下降 |
| 无 ME 且无 CE | 大幅下降 |

#### 与其他 VLA 对比（Q4）

π0.5 **显著优于** π0 和增强版 π0-FAST+Flow，即使后者训练步数增至 300k 仍然如此。这证实了 FAST 离散 token 预训练 + 联合训练配方的优势。

#### 高层推理重要性（Q5）

| 方法 | 相对性能 |
|------|---------|
| 完整 π0.5（高层+低层） | **最佳** |
| 隐式高层（训练含 HL 数据但推理不用） | 第二 |
| 人类 oracle 高层 | 第三（低于完整 π0.5） |
| 无 VI（口头指令） | 显著下降 |
| 无 WD（网络数据） | 显著下降 |
| GPT-4 作为高层策略 | 最差 |

一个重要发现：π0.5 甚至**超越了人类 oracle 高层策略**，说明模型学习到了比人类即时判断更优的子任务规划策略。

### 4.3 消融实验

- **跨具身数据（ME + CE）** 是性能最关键的组成部分，移除任一都导致显著退化
- **网络数据（WD）** 主要提升高层语义推理和对未见物体类别的语言跟随能力
- **口头指令（VI）** 虽然仅占高层移动操作示例的 11%，但对高层策略质量至关重要
- **高层子任务预测（HL）** 即使不在推理时显式使用，仅在训练中包含也能显著提升性能，说明子任务数据隐式地增强了模型的场景理解
- 不同任务对数据源的敏感度不同："物品放入抽屉"依赖 WD 和跨具身数据；"餐具放入水槽"主要依赖跨具身数据；"整理床铺"和"衣物放入篮子"对数据源变化不太敏感

## 五、局限性与展望

- **作者讨论的局限**：
  - 模型仍会犯错：不熟悉的抽屉把手、物理上难以操作的柜门、遮挡导致的部分可观测性问题、高层推理中的重复行为（如反复开关抽屉）
  - 目前仅处理相对简单的自然语言指令，更复杂的偏好和约束需要更丰富的标注数据
  - 使用较短的上下文窗口，缺乏跨房间导航、物体位置记忆等长期记忆能力
- **潜在改进方向**：
  - 更大规模、更多样的数据和更好的联合训练策略
  - 引入更丰富的上下文和记忆机制以应对部分可观测场景
  - 扩展口头指令等新型监督模态，让人类更自然地向机器人传递背景知识
  - 探索更广泛的异构数据源组合

## 六、灵魂三问

1. **它解决了什么问题？** 之前 VLA 模型只在与训练数据匹配的环境中有效，换一个新家就不行；π0.5 通过异构数据联合训练 + 层次化推理，首次实现了端到端学习系统在从未见过的真实家庭中自主完成 10-15 分钟的厨房/卧室清洁任务。

2. **为什么这么做？** 核心选择是"联合训练多种异构数据源（跨具身、网络数据、高层语义、口头指令）"而非"暴力扩大同质移动操作数据"。不这么做的原因：清洁厨房涉及多层次泛化——底层动作需要跨具身数据、语义理解需要网络数据、任务规划需要高层标注——单一数据源无法同时覆盖所有层次，97.6% 的预训练数据不来自目标平台但正是它们赋予了模型泛化能力。

3. **什么证据最有说服力？** Figure 8 的场景扩展实验：104 个场景训练的 π0.5 达到了与"在测试环境上直接训练"的模型相当的性能（~65%），而**不使用联合训练配方、仅用同样 104 个场景数据**的基线远低于此（浅黄色柱）。这直接证明了性能差距来自"数据的多样性"而非"数据的数量"。

## 七、个人总结

1. **最核心的 idea**：π0.5 的核心洞察在于——机器人的开放世界泛化不能仅靠单一数据源的暴力扩展，而需要**将异构多源数据（跨具身机器人、网络视觉语言、高层语义、口头指令）整合到统一的 VLA 框架中联合训练**，并通过层次化推理（高层语义子任务预测 + 低层连续动作生成）实现复杂长时间任务的规划与执行。

2. **最大的优势和不足**：优势在于首次实现了端到端学习系统在全新真实家庭中完成 10-15 分钟的复杂操作任务，且系统设计简洁（同一模型兼任高低层策略，无需额外轨迹规划）。不足在于模型仍对极端新颖的物理交互（如不常见的把手设计）和部分可观测性场景缺乏鲁棒性，且指令理解和上下文记忆能力有限。

3. **对后续研究的启发**：该工作有力地证明了"数据异构性"是实现机器人泛化的关键维度，比简单地增大同质数据规模更有效。层次化推理中"高层训练数据即使不在推理时显式使用也能提升性能"的发现尤其值得关注，暗示了多任务联合训练中存在深层的知识迁移机制。此外，口头指令作为一种轻量但高效的监督信号，为人机协作式的机器人学习开辟了新方向。
