# 基于 Transformer 的可扩展扩散模型

> 原标题：Scalable Diffusion Models with Transformers
> 作者：William Peebles (UC Berkeley), Saining Xie (New York University)
> 发表：ICCV 2023 (arXiv:2212.09748v2)
> 链接：https://arxiv.org/abs/2212.09748
> 项目：https://www.wpeebles.com/DiT

---

## 一句话说清楚

扩散模型生成图像时，核心有一个"去噪引擎"——以前大家默认用 U-Net（一种卷积网络）来做这个引擎，这篇论文说：**换成标准 Transformer 就行，而且效果更好、更好扩展。**

---

## 先搞懂背景：扩散模型是怎么生成图像的？

想象你拿一张清晰照片，往上不断撒噪点，撒 1000 次后就变成了纯噪声——这是**前向过程**，纯数学操作，不需要学习。

反过来，如果有一个神经网络能学会"看一张有噪声的图，预测上面有多少噪声"，那你就可以**从纯噪声开始，一步步减去预测的噪声，最终还原出一张清晰图片**——这是**反向过程**，也是生成图像的过程。

这个负责预测噪声的神经网络就是扩散模型的"去噪引擎"。之前所有人都用一种叫 **U-Net** 的卷积网络做这个引擎。

## 那问题出在哪？

U-Net 用了好几年了，但没人认真回答过一个问题：**U-Net 的卷积结构对扩散模型真的不可或缺吗？**

与此同时，Transformer 已经在 NLP（GPT）、视觉识别（ViT）等几乎所有领域取代了旧架构，展现出了极好的扩展性——模型越大，效果越好，且提升可预测。扩散模型为什么不试试？

这就是 DiT 要回答的问题。

---

## 二、核心贡献

1. **证明了 U-Net 可以被替换**：用标准 ViT 替换 U-Net 做扩散模型骨干，效果更好
2. **找到了最优的条件注入方式**：对比四种方案，adaLN-Zero 以最小的额外计算开销取得最好效果
3. **发现了 scaling law**：生成质量与模型计算量（Gflops）高度相关（相关系数 -0.93），参数量反而不是关键
4. **刷新 SOTA**：ImageNet 256×256 FID 2.27，512×512 FID 3.04，全面超越此前所有模型

## 三、方法原理

### 3.1 整体框架：图像 → 压缩 → 切块 → Transformer 去噪

DiT 的数据流用一句话概括：**先把图片压小，切成小块交给 Transformer 处理，然后把结果拼回去。**

具体来说分三步：

```
原始图像 (256×256×3)
      ↓ VAE 编码器（预训练好的，冻结不动）
潜表示 (32×32×4)          ← 图像被压缩到 1/8 大小，信息保留在 4 个通道里
      ↓ Patchify（切块 + 线性嵌入）
Token 序列 (T 个 token)   ← 类似 ViT 把图片切成小块，每块变成一个向量
      ↓ N 层 DiT Block（Transformer）
Token 序列 (T 个 token)   ← 预测的噪声信息
      ↓ 线性解码 + 重排
噪声预测 (32×32×4)        ← 拼回空间布局，就得到了对当前噪声的预测
```

为什么先压缩再处理？因为直接在 256×256 像素上跑 Transformer 计算量太大。先用 VAE 编码器压缩到 32×32，计算量降了 64 倍，同时关键信息都保留在潜空间里。这就是 Latent Diffusion Model（LDM）的思路，DiT 沿用了它。

**Patchify 如何影响性能**：patch 越小（如 $p=2$），切出来的 token 数 $T=(32/p)^2$ 越多（从 16 个到 256 个），Transformer 要处理的序列越长，计算量越大，但信息粒度更细。论文发现这种"花更多算力处理更细粒度信息"的做法确实能提升生成质量。

### 3.2 关键设计：怎么告诉 Transformer "现在是第几步去噪"和"要生成哪类图"？

扩散模型的去噪引擎需要知道两个额外信息：**时间步** $t$（当前处于去噪的第几步）和**类别标签** $c$（想生成什么类别的图）。关键问题是：这些条件信息怎么注入 Transformer？

论文试了四种方案：

| 方案 | 做法 | 效果 |
|------|------|------|
| **In-Context** | 把 $t$、$c$ 当作额外 token 拼入序列 | 最差，条件信息淹没在图像 token 中 |
| **Cross-Attention** | 每层加一个 Cross-Attention 层来注意条件信息 | 一般，且额外增加 15% 计算量 |
| **adaLN** | 条件信息控制 LayerNorm 的缩放和偏移 $\gamma, \beta$ | 较好，开销最小 |
| **adaLN-Zero** | 在 adaLN 基础上，额外用条件控制残差连接的缩放 $\alpha$，并初始化为零 | **最好** |

**adaLN-Zero 为什么最优？** 关键在于"零初始化"。当 $\alpha=0$ 时，残差连接 $x + \alpha \cdot f(x)$ 退化为 $x + 0 = x$，整个 DiT Block 就是恒等映射——什么都不做。这意味着在训练最开始，一个 28 层的 DiT 等价于直接把输入原样输出。然后在训练过程中，每一层慢慢学会"做一点有用的事"。这种渐进式学习比一上来就让 28 层同时乱搞要稳定得多。

### 3.3 训练细节

训练目标很标准：给模型一张加了噪声的图，让它预测噪声，最小化预测值和真实噪声之间的均方误差：

$$L_{\text{simple}}(\theta) = \|\epsilon_\theta(x_t) - \epsilon_t\|^2_2$$

即模型预测的噪声与实际添加的噪声之间的差距越小，损失越低。

| 设置 | 选择 |
|------|------|
| 优化器 | AdamW，lr=$1 \times 10^{-4}$，恒定不衰减 |
| 批大小 | 256 |
| 数据增强 | 仅水平翻转 |
| EMA 衰减 | 0.9999 |
| VAE | Stable Diffusion 预训练版，冻结 |
| 去噪步数 | 1000 步线性方差调度 |

值得注意的是：DiT 的训练**没有用 warmup、没有正则化**，但训练非常稳定，从未出现 loss spike——这在 Transformer 训练中相当罕见，说明 adaLN-Zero 的初始化策略确实有效。

**推理时的 Classifier-Free Guidance（CFG）**：训练时随机丢弃类别标签（替换为空嵌入），推理时同时预测"有条件"和"无条件"两个噪声，然后放大两者差异：$\hat{\epsilon} = \epsilon_{\text{无条件}} + s \cdot (\epsilon_{\text{有条件}} - \epsilon_{\text{无条件}})$。$s > 1$ 时，生成结果更符合指定类别但多样性降低。

## 四、实验与结果

### 4.1 实验设置

- **数据集**：ImageNet 256×256 和 512×512
- **主指标**：FID-50K（越低越好，衡量生成图与真实图的分布距离）
- **基线**：ADM 系列（U-Net 扩散模型标杆）、LDM 系列（潜空间 U-Net）、StyleGAN-XL、BigGAN-deep

### 4.2 核心发现

**发现一：Transformer 全面超越 U-Net**

| 方法 | FID↓ | Gflops | 骨干类型 |
|------|------|--------|---------|
| ADM | 10.94 | 1120 | 像素空间 U-Net |
| ADM-G + ADM-U | 3.94 | 742 | 像素空间 U-Net |
| LDM-4-G (cfg=1.50) | 3.60 | 104 | 潜空间 U-Net |
| StyleGAN-XL | 2.30 | - | GAN |
| **DiT-XL/2-G (cfg=1.50)** | **2.27** | **118.6** | **潜空间 Transformer** |

DiT 以约 1/10 于 ADM 的计算量实现了更好的生成质量。

**发现二：计算量（Gflops）决定性能，而非参数量**

论文训了 12 种 DiT 变体（4 种尺寸 × 3 种 patch 大小），发现：

- Gflops 与 FID 的相关系数为 **-0.93**（几乎完美的线性关系）
- 关键实验：**固定模型大小（即固定参数量），仅减小 patch 大小**（增加 token 数 → 增加 Gflops）→ FID 显著下降。这说明性能提升不是因为参数多了，而是因为计算量大了
- 相似 Gflops 的不同配置（如 DiT-S/2 约 6G 和 DiT-B/4 约 5.6G）得到了相近的 FID——Gflops 才是决定因素

**发现三：大模型比小模型更"划算"**

同样花 $10^{10}$ Gflops 的总训练预算，是训一个大模型跑少几步好，还是训一个小模型跑多几步好？答案是**大模型更高效**。小模型训再久也追不上大模型——这与 LLM 领域的 scaling law 完全一致。

更极端的是：小模型在推理时增加采样步数（花更多推理算力）也追不上大模型。DiT-L/2 用 1000 步采样消耗 80.7T Gflops，仍不如 DiT-XL/2 用 128 步仅消耗 15.2T Gflops。

### 4.3 条件注入机制的消融

400K 步时 DiT-XL/2 在不同条件注入方式下的 FID：

| 方案 | FID↓ | Gflops |
|------|------|--------|
| In-Context | 35.24 | 119.4 |
| Cross-Attention | 26.14 | 137.6 |
| adaLN | 25.21 | 118.6 |
| **adaLN-Zero** | **19.47** | **118.6** |

adaLN-Zero 以最低的计算开销达到了最好的 FID，与最差的 In-Context 相比 FID 几乎减半。

## 五、局限性与展望

- **局限**：仅验证了类别条件生成（给一个类别标签生成图片），未扩展到文本到图像生成
- **展望**：DiT 可作为 DALL·E 2、Stable Diffusion 等文本到图像模型的骨干——事实上论文发表后，DiT 架构确实被 Sora（视频生成）和 Stable Diffusion 3（文本到图像）等后续工作采用，证明了其通用性

## 六、灵魂三问

1. **它解决了什么问题？** 之前扩散模型被绑死在 U-Net 上，无法享受 Transformer 生态的扩展性红利；现在证明直接用标准 Transformer 替换即可，效果更好还更省算力。

2. **为什么这么做？** 最关键的设计是 adaLN-Zero——用零初始化让每层 Transformer Block 开局为恒等映射，从"什么都不做"逐步学起。不用 Cross-Attention 是因为它多花 15% 算力但效果更差；不用 In-Context 是因为条件信息混在图像 token 里太弱。

3. **什么证据最有说服力？** 固定参数量、仅减小 patch 大小的实验——参数量没变但 Gflops 增加了，FID 显著下降。这直接排除了"效果好是因为参数多"的解释，证明**计算量才是核心驱动力**。

## 七、个人总结

1. DiT 的核心 idea 极简——把 U-Net 换成 ViT，加上 adaLN-Zero 初始化就完事了。真正的贡献不在于架构多复杂，而在于**证明了一个大家默认但从没验证过的假设是错的**（扩散模型必须用 U-Net）。

2. 最大优势是打通了扩散模型与 Transformer 的 scaling law——模型越大效果越好，且可预测。最大不足是论文止步于类别条件生成，没有直接验证文本到图像等更实用的场景。

3. DiT 是"架构统一"趋势的又一个里程碑：NLP 用 Transformer，视觉识别用 Transformer，现在图像生成也用 Transformer。后续的 Sora、SD3 等工作证明了这条路是通的。论文揭示的"Gflops 驱动质量"规律给后续研究提供了清晰的扩展路线——想要更好的生成效果，加大模型或用更小的 patch 就行。
